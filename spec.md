### **Project Specification: Automated Email Processing and AI-Powered Reply Bot**

#### **1. High-Level Description**

This document outlines the requirements for a Python script that automates email handling. The script will monitor a local directory for new email files, use a Large Language Model (LLM) via the Ollama API to understand the email's content, and send an intelligent reply.

#### **2. Core Workflow & Logic**

The script operates in a continuous, single-threaded polling loop.

1.  **Monitor:** The script checks a specified `source_folder` for the presence of any email files every `polling_interval_seconds`. This is a Maildir-type email folder, so all files in the directory will contain exactly one email.
2.  **Process:** It processes files one at a time.
3.  **Extract Data:** For each email file, it performs the following extractions:
    *   **Headers:** The sender's email address (`From`), the `Subject`, `Cc`, and the message id headers: `Message-Id`, `In-Reply-To` and `References`.
    *   **Body:** It extracts the email body content. It MUST prioritize the `text/plain` part of the email. If a plain text part is not available, it MUST fall back to using the `text/html` part.
4.  **Process Attachments:**
    *   It inspects all attachments and compares their file extensions against a configurable `allowed_attachment_extensions` list (e.g., `['.txt', '.md', '.eml']`).
    *   If any attachment has a disallowed extension, the script triggers the "Unsupported Attachment Error" flow (see Section 3).
    *   For all allowed attachments, it reads their text content.
5.  **Construct API Prompt:** It constructs a single text prompt for the Ollama API in the following order and format:
    1.  The content of `ollama_prefix_prompt` from the configuration.
    2.  The text content of each allowed attachment, wrapped in XML-style tags: `<attachment filename="[original_filename]">[attachment_text_content]</attachment>`.
    3.  The extracted email body, wrapped in an XML-style tag: `<email_body>[email_body_content]</email_body>`.
    *   If the email body is empty but attachments are present, the `<email_body>` tag should be omitted.
6.  **Interact with Ollama API:** It sends the complete prompt to the `ollama_api_url`.
7.  **Compose Reply:**
    *   **Subject:** The reply subject line MUST be prefixed with `Re: `, unless the original subject already starts with `Re: `.
    *   **Body:** The reply email body MUST be structured as follows:
        1.  A line containing: `User:`
        2.  The text response generated by the Ollama API.
        3.  A blank line.
        4.  A separator line containing: `Assistant:`
        5.  The original email's body, copied verbatim without any modifications or prepended characters.
8.  **Send Reply:** The composed reply is sent via SMTP to sender and anyone on the CC header.
9.  **Archive:** Upon successful sending, the original email file is moved from the `source_folder` to the `processed_folder`.

#### **3. Error Handling**

*   **API Failure:**
    *   **Retry:** If a call to the Ollama API fails, the script MUST retry the call up to 3 times.
    *   **Notify & Archive:** If all 3 retries fail, the script MUST send an email to the original sender with the error details, then move the email file to the `processed_folder`.
*   **SMTP Failure:**
    *   **Log & Halt:** If the script fails to send an email via SMTP, it MUST log the error.
    *   **Do Not Archive:** It MUST NOT move the email file, ensuring the email is attempted again on the next run.
    *   **Terminate:** The script should exit after an SMTP failure.
*   **Unsupported Attachment Error:**
    *   **Notify:** The script MUST send a reply to the original sender stating which attachments were rejected and listing the allowed types.
    *   **Archive:** After sending the notification, the script MUST move the email file to the `processed_folder` and continue processing other files.

#### **4. Logging**

*   **Level:** Log events at the `INFO` level of detail.
*   **Destination:** Logs should be printed directly to the console.
*   **Content:** Logs should include messages for key actions, such as starting to process a file, successful replies, and all error conditions.

#### **5. Configuration**

All configuration will be managed in a file named `config.py` that contains a single Python dictionary. SMTP credentials MUST be loaded from environment variables, not this file.

**Dictionary Keys:**

| Key | Type | Required | Default | Description |
| :--- | :--- | :--- | :--- | :--- |
| `source_folder` | `str` | Yes | `emails/new` | Path to the folder with new email files. |
| `processed_folder` | `str` | Yes | `emails/cur` | Path to move emails to after successful processing. |
| `polling_interval_seconds`| `int` | Yes | 30 | Time in seconds between checking the source folder. |
| `bot_name` | `str` | Yes | `Assistant` | The display name for the bot in replies (e.g., "Support Bot"). |
| `bot_email` | `str` | Yes | `bot@example.com` | The email address for the bot to send replies from. |
| `ollama_api_url` | `str` | Yes | `http://localhost:11434/api/generate` | The full URL for the Ollama API endpoint. |
| `ollama_model` | `str` | Yes | `llama3` | The specific model to use. |
| `ollama_temperature` | `float` | No | 0.7 | The temperature setting for the model. |
| `ollama_num_ctx` | `int` | No | 4096 | The temperature setting for the model. |
| `ollama_repeat_penalty` | `float` | No | 1.1 | The temperature setting for the model. |
| `ollama_top_k` | `int` | No | 40 | The temperature setting for the model. |
| `ollama_top_p` | `float` | No | 0.9 | The temperature setting for the model. |
| `ollama_min_p` | `float` | No | 0.0 | The temperature setting for the model. |
| `repeat_last_n` | `int` | No | 64 | The temperature setting for the model. |
| `ollama_repeat_penalty` | `float` | No | 1.1 | The temperature setting for the model. |
| `ollama_prefix_prompt` | `str` | No | `You are a helpful email assistant.` | Optional text to prepend to every API prompt. Can be empty. |
| `allowed_attachment_extensions`| `list` | Yes | `['.txt', '.md', '.eml']` | A list of strings for allowed file extensions. |
| `smtp_host` | `str` | Yes | `smtp.example.com` | The SMTP server hostname or IP address. |
| `smtp_port` | `int` | Yes | 25 | The port for the SMTP server. |
| `smtp_use_tls` | `bool` | No | False | `True` or `False` to enable or disable STARTTLS. |
| `smtp_use_ssl` | `bool` | No | False | `True` or `False` to enable or disable a direct SSL connection. |
